## 1. Project Overview

**SugboClean** is a waste management and sanitation web application built for a local government unit (LGU) sanitation office in Cebu, Philippines. "Sugbo" is the ancient name for Cebu. The tagline is _"Keeping Sugbo clean, one pickup at a time."_

The app bridges the gap between residents and the sanitation office by giving both sides a simple, dedicated tool.

---

## 2. User Types

### Residents (No Login Required)

Ordinary Cebuano households and students. They visit the app to:

- Check their pickup schedule
- Report a missed collection
- Track whether their complaint was addressed
- Learn how to sort waste properly
- See hauler contact info and routes
- Subscribe to pickup reminder emails

### LGU Admin (Login Required)

Government employees at the sanitation office. They log in to:

- Review and resolve resident complaints
- Update pickup schedules
- Manage hauler information
- Plot hauler routes on a map
- Monitor trends through analytics charts
- CRUD operations on all data tables

---

## 3. Features (9 Total)

### Resident-Facing Features

1. **Schedule Checker** — Residents select their barangay from a dropdown and see upcoming garbage collection dates, waste type per day (biodegradable, residual, recyclable), and estimated time window.

2. **Missed Pickup Report** — Simple form: barangay, date, waste type, optional photo, optional email, optional description. On submit, system returns a unique code in format `SC-YYYY-NNNN` (e.g., SC-2026-0042). The code is auto-generated by a ServiceNow Business Rule.

3. **Report Status Tracker** — Resident enters their report code (SC-YYYY-NNNN) to see current status: Pending → In Progress → Resolved. Status updates should appear in real time.

4. **Waste Disposal Guide** — Searchable reference page. Items are color-coded by bin type:
   - 🟢 Green = Biodegradable
   - 🔵 Blue = Recyclable
   - ⚫ Black = Residual
   - 🔴 Red = Hazardous

   Residents can search by item name or filter by bin color/type.

5. **Hauler Contact Directory** — List of contracted garbage hauling companies with contact numbers and areas covered.

6. **Hauler Route Map** — Interactive map showing the garbage truck route for a selected hauler. Each barangay stop is a numbered pin in visit order, connected by a dashed line. Clicking a pin shows estimated arrival time and stop status (Not Arrived, Current, Passed).

7. **Pickup Reminder Notification** — On the schedule page, residents enter their email to get day-before reminders about upcoming pickups.

### Admin-Facing Features

8. **LGU Admin Dashboard** — Control center for sanitation staff. After login:
   - View all submitted reports in a table (filterable by barangay, status, date)
   - Update report status (Pending → In Progress → Resolved)
   - CRUD for pickup schedules, hauler companies, route stops, waste items
   - When admin updates a report status, the resident's tracker reflects the change

9. **Basic Admin Analytics** — Charts page inside admin dashboard:
   - **Bar chart:** Which barangays have the most missed pickup reports
   - **Pie chart:** Breakdown by waste type
   - **Line chart:** New reports filed vs. resolved over last 30 days
   - Admins can adjust date range

## 7. UI/UX Guidelines

### Color System

- **Primary:** Green (#16A34A) — represents cleanliness and environment
- **Secondary:** Blue (#2563EB) — for actions and links
- **Status colors:**
  - Pending: Yellow/Amber (#F59E0B)
  - In Progress: Blue (#3B82F6)
  - Resolved: Green (#22C55E)
- **Bin colors:**
  - Biodegradable: Green (#22C55E)
  - Recyclable: Blue (#3B82F6)
  - Residual: Dark/Black (#1F2937)
  - Hazardous: Red (#EF4444)

### Design Principles

1. **Mobile-first** — Most residents will use phones
2. **Simple and fast** — Residents just want quick info, minimal clicks
3. **Bilingual-ready** — Consider Cebuano/English support later
4. **Accessible** — High contrast, large tap targets, clear labels
5. **Offline-friendly** — Show cached schedules if possible

### Navigation

- Resident side: Simple top navbar or bottom tab bar with icons
- Admin side: Sidebar navigation with collapsible menu

### Key Implementation Notes

1. All ServiceNow field names use `u_` prefix (e.g., `u_name`, `u_barangay`, `u_status`)
2. Reference fields store `sys_id` values — when creating/updating records, pass the `sys_id` of the related record
3. The report code (SC-YYYY-NNNN) is auto-generated server-side — don't generate it in the frontend
4. The `sys_id` is the primary key for all records — use it for update/delete operations
5. GET endpoints that return reference fields include both the `sys_id` (in the raw field) and the display value (via `getDisplayValue`)
6. Status flow for reports: Pending → In Progress → Resolved (one direction only)

### What SugboClean is NOT

- Not a social media app or payment platform
- Not a real-time GPS tracking system (route map shows pre-planned static routes)
- No public user accounts (residents never register or log in)
- No messaging system between residents and LGU
- Designed to be as simple and fast as possible
